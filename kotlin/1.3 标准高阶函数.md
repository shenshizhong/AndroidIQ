run() 函数 和 T.run()
```
public inline fun <R> run(block: () -> R): R {
contract {
    callsInPlace(block, InvocationKind.EXACTLY_ONCE)
}
return block()
}

用于条件判断，得到不同的结果：
val index = 3
val num = run {
    when(index){
        0 -> "kotlin"
        1 -> "java"
        2 -> "php"
        3 -> "javaScript"
        else -> "none"
    }
}.length
println("num = $num")
-----------

public inline fun <T, R> T.run(block: T.() -> R): R {
    contract {
        callsInPlace(block, InvocationKind.EXACTLY_ONCE)
    }
    return block()
}

带有当前对象的上下文：
val mTvBtn = findViewById<TextView>(R.id.text)
mTvBtn.run{
    text = "kotlin"
    textSize = 13f
    ...
}

val str = "kotlin"
str.run {
    println( "length = ${this.length}" )
    println( "first = ${first()}")
    println( "last = ${last()}" )
}
```
with 函数
```

public inline fun <T, R> with(receiver: T, block: T.() -> R): R {
    contract {
        callsInPlace(block, InvocationKind.EXACTLY_ONCE)
    }
    return receiver.block()
}

with 与 run 差不多，但run有一个比较好的特点是，在判断null 的时候可读性好一些
val newStr : String? = "kotlin"
with(newStr){
    println( "length = ${this?.length}" )
    println( "first = ${this?.first()}")
    println( "last last = ${this?.last()}" )
}

newStr?.run {
    println( "length = $length" )
    println( "first = ${first()}")
    println( "last = ${last()}" )
}
```

T.apply() 函数
```
与 T.run 的区别是返回了当前对象，T.run 返回执行结果。
T.apply() 相当于建造者模式，可以一直点下去

public inline fun <T> T.apply(block: T.() -> Unit): T {
    contract {
        callsInPlace(block, InvocationKind.EXACTLY_ONCE)
    }
    block()
    return this
}


val mTvBtn = findViewById<TextView>(R.id.text)
mTvBtn.apply{
    text = "kotlin"
    textSize = 13f
    ...
}.apply{
    // 这里可以继续去设置属性或一些TextView的其他一些操作
}.apply{
    setOnClickListener{ .... }
}

```
T.also() 函数
```
T.also 与 T.apply 的区别，主要是在block函数中传入自身对象

public inline fun <T> T.also(block: (T) -> Unit): T {
    contract {
        callsInPlace(block, InvocationKind.EXACTLY_ONCE)
    }
    block(this)
    return this
}

"kotlin".also {
    println("结果：${it.plus("-java")}")
}.also {
    println("结果：${it.plus("-php")}")
}

"kotlin".apply {
    println("结果：${this.plus("-java")}")
}.apply {
    println("结果：${this.plus("-php")}")
}

结果：kotlin-java
结果：kotlin-php

结果：kotlin-java
结果：kotlin-php

```

T.let() 函数
```
主要用于判null

public inline fun <T, R> T.let(block: (T) -> R): R {
    contract {
        callsInPlace(block, InvocationKind.EXACTLY_ONCE)
    }
    return block(this)
}

"kotlin".let {
    println("原字符串：$it")         // kotlin
    it.reversed()
}.let {
    println("反转字符串后的值：$it")     // niltok
    it.plus("-java")
}.let {
    println("新的字符串：$it")          // niltok-java
}

```
T.takeIf() 函数
```
判断条件成立返回自身，否则返回null

public inline fun <T> T.takeIf(predicate: (T) -> Boolean): T? {
    contract {
        callsInPlace(predicate, InvocationKind.EXACTLY_ONCE)
    }
    return if (predicate(this)) this else null
}

val str = "kotlin"
val result = str.takeIf {
    it.startsWith("ko") 
}

println("result = $result")
```

repeat() 函数
```
重复一个函数

public inline fun repeat(times: Int, action: (Int) -> Unit) {
    contract { callsInPlace(action) }

    for (index in 0..times - 1) {
        action(index)
    }
}

repeat(5){
    println("我是重复的第${it + 1}次，我的索引为：$it")
}

```

lazy() 函数
```
Lazy 同步锁，所有线程看到的都是相同的值
有三种模式：SYNCHRONIZED，PUBLICATION，NONE  （SYNCHRONIZED才是线程安全的）
初始过一次就不再执行重复，直接去拿缓存的值

val lazyValue: String by lazy {
     println("computed!")
     "Hello"
}
fun main() {
    println(lazyValue)
    println(lazyValue)  //直接拿缓存的值
}

运行结果为：
computed!
Hello
Hello
```
